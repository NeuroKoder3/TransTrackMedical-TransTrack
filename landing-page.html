The Problem

Transplant candidates get inactivated not because of clinical failures, but operational ones — an expired evaluation nobody tracked, a missing insurance authorization, a coordinator overwhelmed by caseload. The EHR tracks clinical data. UNOS handles allocation. But the operational coordination layer between them is managed with spreadsheets and memory.

What TransTrack Does

TransTrack sits between your EHR and national allocation systems as a dedicated operational risk and readiness platform. It continuously monitors your entire waitlist for the problems that cause delays and inactivations — and surfaces them before they become crises.

Security & Compliance Architecture

Built for healthcare from the foundation — not bolted on after the fact.

Encrypted at Rest

AES-256 via SQLCipher with PBKDF2-HMAC-SHA512 key derivation (256K iterations). OS-level key protection via DPAPI/Keychain.

Offline-First

PHI never leaves the workstation. No cloud dependency, no network transmission in core operating mode. Simplified BAA requirements.

Org Isolation

Multi-tenant architecture with enforced cross-organization data boundaries. Session-bound org context verified by automated tests.

RBAC + Justified Access

Role-based access control with "break the glass" emergency access that requires documented justification.

Electron Hardening

Context isolation enabled, no node integration in renderer, no remote module. Production builds disable dev tools.

Key Rotation

Encryption key rotation with pre-rotation backup, PRAGMA rekey, and complete audit logging of all key operations.
